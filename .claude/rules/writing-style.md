# 文体ルール（Voice Guide）

## 基本トーン
- 親しみやすく、でもロジカル
- 感情論ではなく「構造」で語る
- 自虐的なユーモアを交える
- 読者と同じ目線（上から目線にならない）
- 「やってみたらこうだった」という実験報告のスタンス

## 一人称・語尾
- 一人称: 「僕」
- 語尾: カジュアル
  - 「〜だよ」「〜なんだけど」「〜だと思う」
  - 「〜ってなくない？」「〜じゃん」
- 笑いの表現: 「w」「笑」は適度に使用OK（1記事に3-5回程度）
- 若者言葉: 「マジ」「サイコー」「ヤバい」など適度にOK

## 文章の特徴
1. **結論ファースト**: 最初にバン！と結論を出す
2. **具体的な数字**: 「26歳」「一泊8万円」「関東全ての自治体」
3. **問いかけ**: 読者に考えさせる（「〜と思っていませんか？」）
4. **対比構造**: 「〜ではなく、〜」「〜じゃなくて、〜」
5. **体験ベース**: 抽象論を避け、実際にやったことを語る

## 禁止事項
- ❌ 過度な自慢、マウント
- ❌ 抽象的な精神論（「頑張れば夢は叶う」的な）
- ❌ 上から目線のアドバイス（「〜すべきだ」「〜しなさい」）
- ❌ 堅すぎるビジネス文体（「〜でございます」「〜いたします」）
- ❌ 過剰な謙遜（「大したことないですが」）
- ❌ 長すぎる前置き

## 参考フレーズ（実際の投稿から抽出）

### 結論の出し方
- 「結論、〜した。（再現性は不明）」
- 「僕らなりに試行錯誤して〜したので、もしどなたかの参考になればと思いアウトプットする。」

### 自虐・ユーモア
- 「じゃあ何があったか？何となくの自信！w それだけw」
- 「自分のお金が飛んでくんだよ。笑」
- 「まだ全くお金にはなっていないが笑」

### 振り返り・気づき
- 「振り返ると、全く絶望はなかった。」
- 「今になるとすごくそれを感じていて、〜して正解だったと思う。」
- 「これは、〜していたら絶対になかった未来だと思う。」

### 感情表現
- 「いやサイコーかよ！」
- 「〜ってなって」（驚き・発見）

### 構造的な問いかけ
- 「じゃあ何があったか？」
- 「何で〜か？そして何で〜か？」
- 「つまり、解くべき問いは『〜』」

### 学びのまとめ
- 「〜だと思う。建築でもITでも同じ。」
- 「〜が最も重要なissueだと思う。」

## 文章構成パターン

### パターンA: 結論→展開→学び
```
1. 結論・主張をバン！と出す
2. 「じゃあどうしたか？」で展開
3. 具体的なエピソード
4. 学び・気づきをまとめる
5. 読者への問いかけで締める
```

### パターンB: 問い→体験→解
```
1. 「〜と思っていませんか？」で読者の痛みを突く
2. 「僕も昔は〜だった」で共感
3. 転機となった出来事
4. 具体的にやったこと
5. 結果と学び
```

## トーンの調整ガイド

| シチュエーション | トーン |
|-----------------|--------|
| 失敗談を語る時 | 自虐的に、でも学びを添えて |
| 成功体験を語る時 | 淡々と事実を述べる（自慢にならないよう） |
| アドバイスする時 | 「僕はこうした」という報告形式で |
| 読者を励ます時 | 同じ目線から「一緒に頑張ろう」的な |
