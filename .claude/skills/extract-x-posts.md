# スキル: X投稿抽出

## 呼び出し方
```
claude "記事XXXからX投稿を抽出して"
claude "articles/XXX/final.md からX投稿を10個作って"
```

## 実行手順

### Step 1: 記事読み込み
- `articles/{記事ID}/final.md` を読み込む
- なければ `draft.md` を使用

### Step 2: 投稿生成
- 5タイプ × 2投稿 = 10投稿を生成
- 各投稿は140字以内

### Step 3: 出力
- 出力先: `articles/{記事ID}/x-posts.md`

---

## 投稿タイプ

### Type 1: 刺さるワンライナー（2投稿）
記事内の印象的な一文を抜粋・調整

**特徴**
- 一文で完結
- 読んだ人が「お、なんだこれ」と思う
- 記事を読みたくなるフック

**例**
```
関東全ての自治体に電話した。お宝物件はネットにはない。
```

### Type 2: 数字フック（2投稿）
具体的な数字を含む投稿

**特徴**
- 数字で目を引く
- 事実ベース
- インパクトのある対比

**例**
```
26歳、貯金ほぼゼロ、信用もない。
あったのは「何となくの自信」だけ。

結果、一泊8万円のホテルができた。
```

### Type 3: 問いかけ（2投稿）
読者への質問形式

**特徴**
- 読者に考えさせる
- 共感を呼ぶ痛みポイント
- リプやいいねを誘発

**例**
```
物件探し、ネットだけで終わらせてない？

僕は関東全ての自治体に電話した。
そしたらネットに出てない物件がゴロゴロ見つかった。
```

### Type 4: 逆張り（2投稿）
常識への反論

**特徴**
- 「〜は間違いだ」系
- 議論を呼ぶ（でも炎上しない範囲で）
- 独自の視点

**例**
```
「最初に建築事業を選ぶスタートアップはない」と言われた。

コストとリスクが高すぎるって。
だから選んだ。みんながやらないことに価値がある。
```

### Type 5: 記事リンク付き（2投稿）
記事公開時用

**特徴**
- 要約 + URL
- 公開直後に投稿
- 「読んでみて」的なCTA

**例**
```
【新記事】
26歳で古民家ホテルを作った話。

物件探しから開業まで、全部書いた。
キャッシュなし、信用なし。
あったのは「何となくの自信」だけ。

{URL}
```

## 注意事項

1. **140字以内に収める**（Xの制限）

2. **記事の核心を出しすぎない**
   - 読みたくなるフックは与える
   - でも全部わかっちゃうのはNG

3. **絵文字は基本使わない**
   - 使うとしても1個まで

4. **ハッシュタグは最小限**
   - 多くても2個
   - なくてもOK

5. **改行を活用**
   - 読みやすさ重視
   - 1文ごとに改行もあり
