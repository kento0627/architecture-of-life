# スキル: 対話型記事生成（Interactive Draft）

## 概要
テーマベースで対話しながら記事を生成するスキル。
バックログに縛られず、あらゆるテーマで記事を作成可能。

## 呼び出し方
```
claude "◯◯◯について記事を書きたい"
claude "〜の話を記事にしたい"
claude "最近〜があって、それを発信したい"
```

---

## フロー

```
┌─────────────────────────────────────────────────────────────┐
│  Phase 1: テーマ理解 & 深掘り質問                            │
│  ├─ ユーザーのテーマを受け取る                              │
│  ├─ story-bank.md から関連エピソードを検索                  │
│  ├─ 過去記事から関連トピックを検索                          │
│  └─ 深掘り質問を3-5個投げる                                │
├─────────────────────────────────────────────────────────────┤
│  Phase 2: 回答収集 & 構成提案                               │
│  ├─ ユーザーの回答を収集                                    │
│  ├─ 記事の切り口・構成案を提案                              │
│  └─ ユーザーの承認を得る                                    │
├─────────────────────────────────────────────────────────────┤
│  Phase 3: 下書き生成                                        │
│  ├─ 回答内容を織り込んで下書き生成                          │
│  ├─ 【要追記】マークで追加情報が必要な箇所を明示            │
│  └─ 出力先: articles/{生成されたID}/draft.md               │
└─────────────────────────────────────────────────────────────┘
```

---

## Phase 1: 深掘り質問テンプレート

### 質問の目的
1. **具体的なエピソード**を引き出す
2. **数字・事実**を確認する
3. **感情・気づき**を言語化させる
4. **読者への価値**を明確にする

### 質問パターン

#### A. 体験・ストーリー系のテーマの場合
```
このテーマで記事を書くにあたって、いくつか教えてください！

1. **きっかけ**: これをやろうと思ったきっかけは？何があった？
2. **具体的な行動**: 実際に何をした？どんなステップで進めた？
3. **壁・失敗**: 途中でつまずいたこと、失敗したことは？
4. **数字**: 関連する具体的な数字があれば（期間、金額、回数など）
5. **学び**: 振り返って「これが大事だった」と思うことは？
```

#### B. 考え方・マインド系のテーマの場合
```
このテーマで記事を書くにあたって、いくつか教えてください！

1. **きっかけ**: この考えに至ったきっかけ、体験は？
2. **ビフォー**: 以前はどう考えていた？何が変わった？
3. **具体例**: この考え方を実践した具体的なエピソードは？
4. **反論**: 「でも〜じゃない？」と言われたらどう答える？
5. **読者へ**: この記事を読んだ人に何を持ち帰ってほしい？
```

#### C. How-to・ノウハウ系のテーマの場合
```
このテーマで記事を書くにあたって、いくつか教えてください！

1. **背景**: なぜこのノウハウが必要だった？どんな課題があった？
2. **ステップ**: 具体的にどんな手順でやった？
3. **失敗パターン**: やりがちな失敗、自分も最初やった間違いは？
4. **数字**: 成果を示す具体的な数字は？（期間、コスト、効果など）
5. **再現性**: 他の人が同じことをやるなら、最初に何をすべき？
```

### 関連エピソードの提示
深掘り質問と一緒に、story-bank.mdから関連しそうなエピソードを提示：

```
ちなみに、story-bank.mdにこんな関連エピソードがありました：
- [STORY-001] 関東全自治体への電話営業
- [STORY-003] ホテルがメディアになる

これらを絡めることもできそうですが、どうですか？
```

## Phase 2: 構成提案テンプレート

回答を受けて、構成案を提示：

```
回答ありがとう！こんな構成で書こうと思うけど、どうかな？

---

**タイトル案**
「{タイトル案1}」
「{タイトル案2}」
「{タイトル案3}」

**ターゲット読者**
{想定される読者のPain}

**記事の流れ**
1. リード: {読者の痛みを突く導入}
2. きっかけ: {〜の体験から}
3. 本題: {具体的なストーリー/ノウハウ}
4. 構造: {なぜそうなのかの分析}
5. まとめ: {読者が持ち帰る学び}

**使うエピソード**
- {回答から抽出したエピソード}
- {story-bankからの関連エピソード}

---

この方向でOK？調整したい部分があれば教えて！
```

---

## Phase 3: 下書き生成

構成承認後、下書きを生成。

### 生成時の指示
1. `.claude/rules/writing-style.md` の文体に従う
2. `.claude/rules/article-structure.md` の構成に従う
3. Phase 1-2で得た情報を最大限活用
4. 情報が足りない箇所は【要追記】マーク
5. story-bank.mdから関連キーフレーズを引用

### 出力先
```
articles/{YYYYMMDD}-{slug}/draft.md
```
例: `articles/20260115-diy-hotel/draft.md`

### 同時生成
下書きと一緒に、story-bank.mdへの追記案も生成：

```markdown
## story-bank.md への追記案

### [STORY-XXX] {タイトル}
- **時期**: {回答から}
- **状況**: {回答から}
- **やったこと**: {回答から}
- **結果**: {回答から}
- **学び**: {回答から}
- **キーフレーズ**: 「{印象的なフレーズ}」

↑ この内容を story-bank.md に追加しますか？ (y/n)
```

## 対話例

### ユーザー
```
最近、物件の仕入れがうまくいかなくて試行錯誤してるんだけど、その話を記事にしたい
```

### Claude（Phase 1）
```
物件仕入れの試行錯誤、いいテーマですね！記事にするにあたって、いくつか教えてください。

1. **現状の課題**: 具体的にどんな壁にぶつかってる？（競合が多い、情報が来ない、など）
2. **試したこと**: これまでどんな方法を試した？うまくいったこと、いかなかったことは？
3. **数字**: 何件くらいアプローチして、何件成約できてる？期間は？
4. **気づき**: 試行錯誤の中で「これ大事だな」と思ったことは？
5. **今後**: 次に試そうとしていることは？

ちなみに、story-bank.mdにこんな関連エピソードがありました：
- [STORY-001] 関東全自治体への電話営業（いすみの物件を見つけた時の話）

これと比較して「あの時と今は何が違う」みたいな切り口もありかも。どう思う？
```

### ユーザー
```
1. 都内の民泊用物件を探してるんだけど、競合が多すぎて全然情報が来ない
2. 不動産屋に片っ端から電話、知り合いに紹介依頼、SNSでの発信を試した。SNSは反応ゼロ、不動産屋は10件に1件くらいしかまともに相手してくれない
3. 3ヶ月で50件くらいアプローチして、内見まで行けたのは5件、成約は0
4. いすみの時と違って「急いでない」のがダメなのかも。あと、不動産屋との関係構築が足りてない
5. 特定の不動産屋と深く付き合うことを試そうとしてる
```

### Claude（Phase 2）
```
なるほど、めちゃくちゃリアルな試行錯誤ですね。こんな構成で書こうと思うけど、どうかな？

---

**タイトル案**
1. 「3ヶ月50件アプローチして成約ゼロ。都内物件探しで学んだ『急いでない問題』」
2. 「いすみでは成功した方法が、都内では通用しなかった話」
3. 「不動産屋に相手にされない問題。関係構築の重要性に気づいた3ヶ月」

**ターゲット読者**
- 民泊や不動産投資を始めたいが、物件が見つからず悩んでいる人
- 「とりあえず数打てば当たる」と思っている人

**記事の流れ**
1. リード: 「物件探し、数打てば当たると思ってませんか？」
2. いすみとの比較: あの時はうまくいった方法
3. 都内での壁: 3ヶ月50件の現実
4. 気づき: 「急いでない」問題と関係構築の重要性
5. 次の一手: 特定の不動産屋と深く付き合う戦略

**使うエピソード**
- 今回の回答（3ヶ月50件→成約0）
- STORY-001との対比（いすみでの成功体験）

---

この方向でOK？「成約ゼロ」をそのまま出すか、もう少しぼかすかは調整できます。
```

### ユーザー
```
いいね！成約ゼロはそのまま出してOK。リアルな方がいい
```

### Claude（Phase 3）
```
了解！下書きを生成します。

[下書き生成中...]

---

articles/20260115-tokyo-property-search/draft.md を作成しました。

また、story-bank.md への追記案も作りました：

### [STORY-015] 都内物件探し3ヶ月の試行錯誤
- **時期**: 2025-2026年
- **状況**: 都内民泊用物件の仕入れ
- **やったこと**: 不動産屋への電話、知り合い紹介依頼、SNS発信
- **結果**: 3ヶ月で50件アプローチ→内見5件→成約0
- **学び**: 「急いでない」のがダメ。関係構築が足りていなかった
- **キーフレーズ**: 「数打てば当たる、は都内では通用しなかった」

この内容を story-bank.md に追加しますか？
```

## 補足: バックログとの連携

この対話型フローで生成した記事は、後からバックログに追加してもOK：

```
claude "今日書いた記事をバックログに追加して"
```

→ `backlog/article-ideas.md` に追記
